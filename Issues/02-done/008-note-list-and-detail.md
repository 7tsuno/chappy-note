# 008: ノート一覧・詳細ビュー実装

## 目的
- タグフィルタと最新順ソートでノートを探せる一覧画面、および本文・メタ情報を参照できる詳細ビューを実装する。

## スコープ
- ノート一覧テーブル/カード、タグフィルタUI、検索入力。
- 詳細ビューでタイトル、タグ、本文、引用リンク表示。
- 「編集」「チャットで質問」などのCTA配置。
- Storybookで一覧/フィルタ/検索ゼロ件/詳細ビューを確認できるStoryを整備し、レビューのたびに更新する。

## 前提Issue
- 001: モノレポ基盤構築
- 002: ドメインモデルと共通ユーティリティ定義
- 006: Reactウィジェットシェル実装

## 完了条件
- `notes.list` / `notes.search` のレスポンスをstructuredContent経由で受け取り、一覧にレンダリングできる。
- タグをクリックするとフィルタが適用され、結果が再描画される。
- 詳細ビューで「編集」を押すとIssue007のエディタに遷移する。
- Storybookで一覧・詳細・空状態の画面が動作し、ユーザーがタイミングよくレビューできる。

## 作業リスト
1. 一覧用コンポーネントを実装し、タグバッジ・更新日時・抜粋を表示する。
2. タグフィルタUI（複数選択可能）と検索入力欄を配置し、ユーザー操作で`notes.search`ツールを呼ぶ。
3. 検索結果が0件の際のEmpty stateとローディングIndictorを用意。
4. 詳細ビューでは本文をMarkdown表示（必要なら`react-markdown`利用）し、タグ編集UIへの導線を明示。
5. 「チャットで質問」ボタンから`sendFollowUpMessage`を呼び、選択ノートの要約＋引用を含むメッセージテンプレを送信。
6. Storybookに一覧/詳細のStoriesを追加し、レビューで指摘を反映するワークフローを定義。
